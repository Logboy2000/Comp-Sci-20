CSE 2910 - Project B

Overview

A top-down shooter game made in the Godot Engine with progressively harder difficulty, an upgrade system, and one "super move" that charges over time.

Inspirations:

Enter the Gungeon
Windowkill

Upgrades:

  • Move speed
  • Fire rate
  • Multishot
  • Piercing
  • Max HP
  • Healing
  • Explosive bullets/splash damage

Controls:

  • WASD: Move
  • Spacebar: Shop
  • Left Click: Shoot
  • Right Click: Super Move

Modules Incorporated:

  • 2110: Procedural Programming 1
  • 2120: Data Structures 1

Required Learning:

  • Understanding of Godot Engine systems
  • Basic Object Oriented Programming
  • GDScript Syntax

Expected Outcomes

Multiple Interconnected Systems

  • Game Systems: The game should have seamless integration between various systems such as player movement, shooting, enemy AI, upgrades, and in-game economy. These systems should interact with each other in a way that keeps the player engaged while providing the necessary challenge.
  • Progressive Difficulty: As the player progresses, the difficulty should naturally increase, making the game more challenging. This includes more enemies, faster enemy spawn rates, and potentially tougher enemies.

Bug-Free

  • Testing and Debugging: The game should go through testing to ensure that all features work as expected without errors.

Global Godot Nodes

  • Room Transitions: The use of global nodes to allow for smooth transitions between different levels or rooms in the game.
  • Settings Management: Players should be able to adjust game settings (graphics, sound, fullscreen, etc.)
  • Debugging: Global nodes can help with debugging by displaing information about various game systems similar to the Minecraft debug menu

Evaluation Breakdown

30% - Basic Shooting and Player Movement, Enemy Killing, Collision

  • Player Movement: The player should be able to move smoothly in all directions with the WASD keys.
  • Shooting Mechanism: The basic shooting mechanics should be implemented, allowing the player to shoot in the direction of the cursor with a left-click.
  • Enemy Killing and Collision Detection: Enemies should be easily killed when hit by bullets, and collision detection should be precise.

50% - Currency, Upgrades, Random Enemy Spawns

  • Currency System: A currency system will allow players to collect resources by defeating enemies, used to purchase upgrades.
  • Upgrades: The player will be able to spend currency to upgrade various aspects of their character.
  • Random Enemy Spawning: Enemies will spawn at random intervals and locations, keeping gameplay dynamic.

80% - Multiple Enemy Types, Difficulty Scaling, Game Menu, Upgrade System

  • Multiple Enemy Types: Several types of enemies are introduced to keep gameplay varied.
  • Difficulty Scaling: The game will progressively become harder over time by increasing spawn rate, health, and damage of enemies.
  • Game Menu System: The game should include a functional menu system with options like starting a new game, exiting, pausing, etc.
  • Upgrades and Pricing System: Each upgrade will increase in price after each purchase, creating progression.

100% - Visual Effects, Audio, Randomized Shop Upgrades

  • Visual Effects: The game will feature cool visual effects like shaders, camera shake, and particle systems.
  • Audio: The game includes fitting background music and sound effects.
  • Randomized Shop Upgrades: The in-game shop will randomly cycle through available upgrades.
